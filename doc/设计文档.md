# VideoClipper Web App 最简设计

## 目标

为 `auto_clipper` Python 工具包提供一个最小可用的 Web App，让外部测试人员可以上传视频、选择语言、一键自动裁剪，并在页面上查看处理日志、最终字幕 TSV 和生成的子视频列表。

本版本优先验证核心功能链路，不做复杂账号系统、任务队列、自动清理、多人协同管理和高级配置。

## 功能范围

### 必须支持

1. 上传一个视频文件，单文件大小不超过 300 MB。
2. 选择视频语言：中文或英文。
3. 点击按钮后自动执行字幕生成和视频裁剪。
4. 前端轮询后台接口，读取任务日志文件并展示裁剪进度。
5. 任务完成后展示最终字幕 TSV 内容，以及对应的子视频列表。
6. 点击子视频后可以在浏览器中播放。
7. 每个上传任务的数据放在独立子目录中，由人工后续清理。

### 暂不支持

1. 用户登录、权限管理和任务归属。
2. 多机部署和分布式任务队列。
3. 在线删除任务数据。
4. 在线编辑字幕、合并片段或重新裁剪。
5. 自动清理历史任务。
6. 自定义 WhisperX、FFmpeg、LLM 等高级参数。

## 技术选型

### 后端

使用 `FastAPI` 实现最小后端服务：

1. 支持文件上传和静态文件访问。
2. 通过 Python API 调用 `auto_clipper` 工具包。
3. Web App 统一维护一个 `config.yaml`，所有任务共用该配置。
4. 将 Python API 执行过程中的 stdout/stderr 和应用日志追加写入任务日志文件。
5. 通过 JSON 文件维护任务状态。

### 前端

使用服务端模板或单个静态 HTML 页面加少量原生 JavaScript：

1. 上传表单：视频文件、语言选择、开始按钮。
2. 任务详情页：状态、日志区域、字幕表格、视频列表。
3. 使用 `fetch` 定时轮询任务状态和日志。

### 工具包调用方式

最小版本使用 Python API 调用工具包，不直接拼接命令行参数：

```python
from copy import deepcopy
from auto_clipper import load_config, run_clipper

base_config = load_config("config/video_clipper.yaml")
task_config = deepcopy(base_config)
task_config.subtitle.language = task.language

run_clipper(
    task.input_path,
    task.subtitle_path,
    video_clip_dir=task.clips_dir,
    mode=app_settings.mode,
    config=task_config,
)
```

说明：

1. Web App 提供统一配置文件 `config/video_clipper.yaml`。
2. 上传页面选择的语言只覆盖 `config.subtitle.language`，目前允许 `zh` 和 `en`。
3. 默认处理模式使用 `duration`，避免最小版本依赖 LLM API 配置。
4. `video_clip_dir` 固定指向当前任务的 `output/clips`，确保切片结果留在任务目录内。
5. 后续如需启用 LLM 语义合并，可将 Web App 配置中的默认模式改为 `llm`，并在同一个配置文件中补充 `llm`、`segmentation` 和 `llm_merge` 配置。

### 统一配置文件

配置文件路径：

```text
config/video_clipper.yaml
```

最小配置示例：

```yaml
subtitle:
  model_name: large-v3
  device: auto
  compute_type: auto
  language: zh
  batch_size: 16
  align: true
  return_char_alignments: false

output:
  timestamp_format:
  include_header: false
  write_word_json: true
  word_json_suffix: _words

segmentation:
  percentile: 95
  output_suffix: _duration

video_split:
  ffmpeg_path: ffmpeg
  ffprobe_path: ffprobe
  output_extension:
  overwrite: true
  copy_codecs: true
  reencode_with_source_codecs: true
  output_encoding_args: []
  index_width: 3
  max_text_length: 80
```

配置原则：

1. Web App 启动时读取并校验 `config/video_clipper.yaml`。
2. 每个任务执行时深拷贝一份配置对象，避免并发任务互相修改。
3. 语言由页面选择覆盖，其他配置不暴露给测试人员。
4. 如果配置文件缺失或加载失败，服务启动失败，不允许进入半可用状态。

## 目录结构

建议项目结构：

```text
VideoClipper-WebApp/
  app/

    settings.py
    templates/
      index.html
      task.html
    static/
      app.js
      app.css
  config/
    video_clipper.yaml
  data/
    tasks/
      {task_id}/
        upload/

        output/
          subtitles.tsv
          clips/
            001_xxx.mp4
            002_xxx.mp4
        logs/
          run.log
        task.json
  doc/
    设计文档.md
```

`data/tasks/{task_id}` 是单个任务的完整数据边界。测试阶段不提供自动删除能力，直接由维护者手动删除对应任务目录。

## 任务目录规范

每次上传成功后生成一个唯一 `task_id`，推荐格式：

```text
YYYYMMDD_HHMMSS_{8位随机字符串}
```

示例：

```text
20260618_153012_a1b2c3d4
```

单任务目录：

```text
data/tasks/20260618_153012_a1b2c3d4/
  upload/input.mp4
  output/subtitles.tsv
  output/clips/
  logs/run.log
  task.json
```

`task.json` 示例：

```json
{
  "task_id": "20260618_153012_a1b2c3d4",
  "status": "running",
  "language": "zh",
  "mode": "duration",
  "config_path": "config/video_clipper.yaml",
  "original_filename": "demo.mp4",
  "input_path": "data/tasks/20260618_153012_a1b2c3d4/upload/input.mp4",
  "subtitle_path": "data/tasks/20260618_153012_a1b2c3d4/output/subtitles.tsv",
  "clips_dir": "data/tasks/20260618_153012_a1b2c3d4/output/clips",
  "log_path": "data/tasks/20260618_153012_a1b2c3d4/logs/run.log",
  "created_at": "2026-06-18T15:30:12+08:00",
  "started_at": "2026-06-18T15:30:13+08:00",
  "finished_at": null,
  "error": null
}
```

任务状态枚举：

| 状态 | 含义 |
| --- | --- |
| `uploaded` | 文件已上传，尚未开始处理 |
| `running` | 后台正在执行裁剪 |
| `succeeded` | 处理完成，可查看结果 |
| `failed` | 处理失败，页面展示错误和日志 |

## 后端接口设计

### 首页

```http
GET /
```

返回上传页面。

### 创建任务

```http
POST /api/tasks
```

请求：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `file` | File | 是 | 视频文件，最大 300 MB |
| `language` | string | 是 | `zh` 或 `en` |

处理逻辑：

1. 校验语言只能是 `zh` 或 `en`。
2. 校验上传文件大小不超过 300 MB。
3. 创建 `data/tasks/{task_id}` 子目录。
4. 保存上传文件到 `upload/input.{ext}`。
5. 初始化 `task.json` 和 `logs/run.log`。
6. 启动后台任务，通过 Python API 执行 `run_clipper(...)`。
7. 返回 `task_id` 和任务详情页地址。

响应：

```json
{
  "task_id": "20260618_153012_a1b2c3d4",
  "detail_url": "/tasks/20260618_153012_a1b2c3d4"
}
```

### 任务详情页

```http
GET /tasks/{task_id}
```

返回任务详情页面。

### 查询任务状态

```http
GET /api/tasks/{task_id}
```

响应：

```json
{
  "task_id": "20260618_153012_a1b2c3d4",
  "status": "running",
  "language": "zh",
  "error": null,
  "created_at": "2026-06-18T15:30:12+08:00",
  "started_at": "2026-06-18T15:30:13+08:00",
  "finished_at": null
}
```

### 查询任务日志

```http
GET /api/tasks/{task_id}/log?offset=0
```

响应：

```json
{
  "offset": 1024,
  "content": "log text..."
}
```

前端保存上一次返回的 `offset`，下次轮询时只读取增量日志。最小版本也可以直接返回完整日志，但日志较长时页面会越来越慢，因此推荐支持 offset。

### 查询任务结果

```http
GET /api/tasks/{task_id}/result
```

任务成功后返回：

```json
{
  "task_id": "20260618_153012_a1b2c3d4",
  "subtitle": {
    "path": "/api/tasks/20260618_153012_a1b2c3d4/subtitle",
    "rows": [
      {
        "index": 1,
        "start": "00:00:00.000",
        "end": "00:00:02.250",
        "text": "你好，世界"
      }
    ]
  },
  "clips": [
    {
      "filename": "001_你好，世界.mp4",
      "url": "/media/tasks/20260618_153012_a1b2c3d4/output/clips/001_%E4%BD%A0%E5%A5%BD.mp4"
    }
  ]
}
```

说明：

1. 后端读取 `output/subtitles.tsv`，按 TSV 解析前三列。
2. `clips` 按文件名排序。
3. 页面中用 HTML5 `<video controls>` 播放选中的子视频。

### 下载或查看字幕文件

```http
GET /api/tasks/{task_id}/subtitle
```

返回原始 `subtitles.tsv` 文件，便于测试人员下载或复制。

### 静态媒体访问

```http
GET /media/tasks/{task_id}/output/clips/{filename}
```

用于播放生成的子视频文件。必须校验 `task_id` 和文件名，避免路径穿越。

## 后台任务执行流程

1. 上传完成后，后端创建任务目录和 `task.json`。
2. 后端将状态更新为 `running`。
3. 后台线程或任务执行器加载统一配置文件 `config/video_clipper.yaml`。
4. 深拷贝配置对象，并将页面选择的语言写入 `task_config.subtitle.language`。
5. 调用 `run_clipper(...)` 执行字幕生成和视频切片。
6. Python API 执行过程中的 stdout/stderr 和应用日志统一追加写入 `logs/run.log`。
7. `run_clipper(...)` 正常返回时，将状态更新为 `succeeded`。
8. `run_clipper(...)` 抛出异常时，将状态更新为 `failed`，并写入 `error` 字段。

伪代码：

```python
import contextlib
import logging
from copy import deepcopy

from auto_clipper import load_config, run_clipper


base_config = load_config(settings.video_clipper_config_path)


def run_task(task):
    update_status(task, "running")

    task_config = deepcopy(base_config)
    task_config.subtitle.language = task.language

    logger = logging.getLogger("videoclipper.task")
    file_handler = logging.FileHandler(task.log_path, encoding="utf-8")
    logger.addHandler(file_handler)

    try:
        with open(task.log_path, "a", encoding="utf-8") as log_file:
            with contextlib.redirect_stdout(log_file), contextlib.redirect_stderr(log_file):
                logger.info("Task %s started", task.task_id)
                run_clipper(
                    task.input_path,
                    task.subtitle_path,
                    video_clip_dir=task.clips_dir,
                    mode=settings.video_clipper_mode,
                    config=task_config,
                )
                logger.info("Task %s succeeded", task.task_id)

        update_status(task, "succeeded")
    except Exception as exc:
        logger.exception("Task %s failed", task.task_id)
        update_status(
            task,
            "failed",
            error=str(exc),
        )
    finally:
        logger.removeHandler(file_handler)
        file_handler.close()
```

## 前端页面设计

### 上传页

页面元素：

1. 文件选择框。
2. 语言选择：中文、英文。
3. 开始裁剪按钮。
4. 上传中状态提示。
5. 上传失败错误提示。

交互：

1. 用户选择文件和语言。
2. 点击开始裁剪。
3. 前端通过 `POST /api/tasks` 上传。
4. 上传成功后跳转到 `/tasks/{task_id}`。

### 任务详情页

页面元素：

1. 当前任务状态。
2. 日志展示区域。
3. 结果区域：任务成功后显示。
4. 字幕 TSV 表格。
5. 子视频列表。
6. 视频播放器。

轮询逻辑：

1. 每 2 秒请求一次 `/api/tasks/{task_id}`。
2. 每 2 秒请求一次 `/api/tasks/{task_id}/log?offset={offset}`。
3. 当任务状态变为 `succeeded` 时，请求 `/api/tasks/{task_id}/result` 并停止状态轮询。
4. 当任务状态变为 `failed` 时，停止状态轮询，保留日志展示。

## 文件大小和输入校验

后端必须做最终校验，前端校验只作为体验优化。

校验规则：

1. 文件大小不得超过 `300 * 1024 * 1024` 字节。
2. 语言只允许 `zh` 和 `en`。
3. 上传文件名不直接作为存储文件名，避免特殊字符问题。
4. 可接受的视频后缀建议先限制为：`.mp4`、`.mov`、`.mkv`、`.webm`、`.avi`。
5. 所有路径必须基于 `data/tasks/{task_id}` 拼接并校验解析后的真实路径在任务目录下。

## 并发策略

最小版本先采用单机本地执行。

建议限制同时运行任务数：

1. 第一版可以只允许 1 个任务同时运行，避免 WhisperX/PyTorch/FFmpeg 抢占 GPU 或内存。
2. 如需排队，可维护一个简单内存队列。
3. 服务重启后不恢复正在运行的任务，重启前正在执行的任务可标记为 `failed` 或由人工清理。

如果希望实现更稳妥的任务排队，后续可引入 Redis Queue、Celery 或 Dramatiq。

## 配置项

建议通过环境变量配置：

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `VIDEOCLIPPER_TASK_ROOT` | `data/tasks` | 任务数据根目录 |
| `VIDEOCLIPPER_MAX_UPLOAD_MB` | `300` | 最大上传大小 |
| `VIDEOCLIPPER_CONFIG_PATH` | `config/video_clipper.yaml` | Web App 统一使用的工具包配置文件 |
| `VIDEOCLIPPER_MODE` | `duration` | 默认处理模式 |
| `VIDEOCLIPPER_MAX_RUNNING_TASKS` | `1` | 最大同时运行任务数 |

## 部署方式

最小部署命令：

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -e /path/to/VideoClipper
python -m pip install fastapi uvicorn python-multipart jinja2
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

运行前要求：

1. 机器已安装 FFmpeg 和 ffprobe。
2. Python 环境能正常导入 `auto_clipper` 包。
3. WhisperX/PyTorch 依赖已经安装并可用。
4. `config/video_clipper.yaml` 已存在且能被 `auto_clipper.load_config(...)` 正常加载。
5. `data/tasks` 所在磁盘有足够空间。

## 最小实现步骤

1. 创建 FastAPI 项目骨架。
2. 实现上传接口和 300 MB 限制。
3. 实现任务目录创建和 `task.json` 状态写入。
4. 实现统一配置文件加载和启动时校验。
5. 实现后台任务通过 Python API 执行 `run_clipper(...)` 并写日志。
6. 实现任务状态、日志和结果查询接口。
7. 实现上传页和任务详情页。
8. 使用一个短视频手动验证中文、英文两种语言。

## 验收标准

1. 上传 300 MB 以内的视频后，页面能跳转到任务详情页。
2. 语言选择为中文时，后台任务使用 `task_config.subtitle.language = "zh"`。
3. 语言选择为英文时，后台任务使用 `task_config.subtitle.language = "en"`。
4. 任务执行期间，日志区域能持续更新。
5. 任务完成后，页面能展示 `subtitles.tsv` 的内容。
6. 任务完成后，页面能列出 `output/clips/` 下的视频文件。
7. 点击任意子视频后，浏览器能播放对应文件。
8. 每个任务的数据都只出现在自己的 `data/tasks/{task_id}` 子目录下。
9. 删除某个任务目录不会影响其他任务。
